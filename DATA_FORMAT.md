# PAPO Data Format

This document defines the JSON and JSONL artifacts used by the PAPO pipeline.

## Conventions

- `.json` contains one JSON object, usually an audit or summary.
- `.jsonl` contains one independent JSON object per line.
- IDs are stable join keys between pipeline stages.
- Paths point to original screenshots and XML files; image bytes are not copied.
- Files containing `smoke` are validation artifacts and must not be used as final training data.

## Data Flow

```text
FingerTip raw episode
  -> proactive/execution tasks
  -> PAPO episodes and steps
  -> counterfactual trees
  -> action advantages
  -> DPO preference pairs
```

| Artifact | Generated by | Consumed by |
|---|---|---|
| `proactive_suggestion*.jsonl` | `proactive_suggestion.py` | Intent prediction evaluation |
| `personalized_execution*.jsonl` | `personalized_execution.py` | Personalized execution evaluation |
| `papo_episodes.jsonl` | `00_build_papo_from_raw.py` | Auditing and episode joins |
| `papo_steps.jsonl` | `00_build_papo_from_raw.py` | `01_build_papo_trees.py` |
| `papo_trees.jsonl` | `01_build_papo_trees.py` | Reward propagation and tree summary |
| `papo_advantages.jsonl` | `02_propagate_papo_advantages.py` | `03_export_papo_dpo_pairs.py` |
| `papo_dpo_pairs.jsonl` | `03_export_papo_dpo_pairs.py` | DPO training |
| `papo_listwise_targets.jsonl` | `09_run_config_pipeline.py` | PAPO target-policy distillation |
| `papo_raw_audit.json` | `00_build_papo_from_raw.py` | Data-quality inspection |
| `papo_tree_summary.json` | `04_summarize_papo_trees.py` | Tree-quality inspection |
| `global_dataset_audit.json` | `05_audit_fingertip_dataset.py` | Extraction readiness inspection |

## Raw FingerTip Files

### `survey_result.json`

One file describes one episode.

```json
{
  "user_id": "1",
  "time": "20250309_133115",
  "scenario": "学校",
  "app": "com.huawei.deskclock",
  "intentDescription": "打开闹钟，设置一个二十分钟后的闹铃"
}
```

| Field | Meaning |
|---|---|
| `user_id` | User identity used for personalization and history retrieval |
| `time` | Episode time used to enforce strict-past retrieval |
| `scenario` | User context, such as home, school, or workplace |
| `app` | Android package/activity identifier |
| `intentDescription` | Ground-truth user instruction |

### `action.jsonl`

Each line records one observed user action.

```json
{"1741498279844.jpg": "click(coordinates=(336, 1267), content='蓝色加号按钮')"}
```

The screenshot filename timestamp joins the action with a screenshot and XML
observation.

## Task Records

### Proactive Suggestion

```json
{
  "task_id": "suggestion__1__20250309_133115",
  "task_type": "proactive_suggestion",
  "input": {
    "user_id": "1",
    "time": "20250309_133115",
    "scenario": "学校",
    "user_profile": {},
    "previous_intents": [],
    "initial_screenshots": []
  },
  "target": {
    "intent": "打开闹钟，设置一个二十分钟后的闹铃",
    "app": "com.huawei.deskclock",
    "intent_class": "闹钟/计时"
  },
  "metadata": {
    "history_policy": "same_user_strictly_before_target_time",
    "target_is_hidden_from_input": true
  }
}
```

`input` is safe model context. `target` is supervision or evaluation data and
must not be included in the model prompt.

### Personalized Execution

```json
{
  "task_id": "execution__1__20250309_133115",
  "task_type": "personalized_execution",
  "input": {
    "instruction": "打开闹钟，设置一个二十分钟后的闹铃",
    "initial_screenshot": "...",
    "initial_xml": "...",
    "same_user_action_reference": {},
    "cross_user_action_reference": {}
  },
  "target": {
    "actions": ["click(...)", "finished()"],
    "intent_class": "闹钟/计时"
  },
  "metadata": {
    "target_actions_are_evaluation_only": true
  }
}
```

| Field | Use |
|---|---|
| `same_user_action_reference` | Personalization context from an earlier similar episode |
| `cross_user_action_reference` | Counterfactual context from another user's earlier episode |
| `target.actions` | Ground-truth trajectory used only for supervision or evaluation |

Both references must be strictly earlier than the target episode.

## PAPO Episode Record

`papo_episodes.jsonl` contains one row per processed episode.

```json
{
  "episode_id": "1__20250309_133115",
  "user_id": "1",
  "time": "20250309_133115",
  "app": "com.huawei.deskclock",
  "scenario": "学校",
  "intent": "打开闹钟，设置一个二十分钟后的闹铃",
  "num_actions": 6,
  "num_steps": 6,
  "num_observations": 6,
  "episode_path": "..."
}
```

`episode_id` is the join key used by task, step, tree, advantage, and DPO
records.

## PAPO Step Record

`papo_steps.jsonl` is the main normalized transition dataset. One row represents
one state-action-next-state transition.

```json
{
  "papo_step_id": "1__20250309_133115__0000",
  "episode_id": "1__20250309_133115",
  "user_id": "1",
  "step_index": 0,
  "chronological_rank": 0,
  "intent": "打开闹钟，设置一个二十分钟后的闹铃",
  "state_key": "com.huawei.deskclock|ui:...",
  "action": "select:蓝色加号按钮",
  "next_state_key": "com.huawei.deskclock|ui:...",
  "valid_observation": true,
  "has_next_state": true,
  "is_terminal": false
}
```

### Identity and Ordering

| Field | Meaning |
|---|---|
| `papo_step_id` | Stable step identifier |
| `episode_id` | Parent episode |
| `user_id` | User identity |
| `step_index` | Position inside the episode |
| `chronological_rank` | Global chronological ordering used for strict-past history |
| `time` | Original episode timestamp |

### State and Observation

| Field | Meaning |
|---|---|
| `state_key` | Stable semantic UI-state identifier |
| `screenshot_path` | Current screenshot |
| `xml_path` | Current accessibility tree |
| `object_tokens` | Compact semantic tokens extracted from the UI |
| `valid_observation` | Whether both screenshot and XML are available |

### Action

| Field | Meaning |
|---|---|
| `raw_action` | Original FingerTip action string |
| `action` | Normalized semantic action label |
| `action_type` | Raw action type such as click, type, or scroll |
| `semantic_verb` | Normalized verb such as select, submit, or input |
| `object_role` | UI target role |
| `object_text` | UI target text |
| `target_bounds` | Target rectangle |
| `action_confidence` | Confidence of action-to-UI grounding |

### Transition

| Field | Meaning |
|---|---|
| `next_state_key` | State after the action |
| `next_screenshot_path` | Next screenshot |
| `next_xml_path` | Next accessibility tree |
| `has_next_state` | Whether a transition exists |
| `is_terminal` | Whether the step ends the episode |

## Counterfactual Tree Record

`papo_trees.jsonl` contains one tree per selected root step.

```json
{
  "tree_id": "papo_tree__1__20250309_133115__0000",
  "root_step_id": "1__20250309_133115__0000",
  "user_id": "1",
  "episode_id": "1__20250309_133115",
  "target_actions": ["select:加号", "scroll:up"],
  "nodes": [],
  "leaves": [],
  "metadata": {}
}
```

### Tree Node

```json
{
  "node_id": "node_00000",
  "proxy_step_id": "1__20250309_133115__0000",
  "depth": 0,
  "state_key": "...",
  "prefix_actions": [],
  "candidates": [
    {
      "action": "select:加号",
      "source": "observed_path",
      "support": 1,
      "transition_step_ids": ["..."],
      "example_step_id": "...",
      "valid": true
    }
  ]
}
```

Candidate sources:

| Source | Meaning |
|---|---|
| `observed_path` | Action actually performed in the target trajectory |
| `same_user` | Action retrieved from the same user's earlier trajectory |
| `cross_user` | Action retrieved from another user's earlier trajectory |

### Tree Leaf

```json
{
  "leaf_id": "leaf_...",
  "actions": ["select:加号", "scroll:up"],
  "path": [],
  "target_actions": ["select:加号", "scroll:up"],
  "user_score": 0.72,
  "r_task": 1.0,
  "r_user": 1.0,
  "leaf_weight": 2.0
}
```

| Field | Meaning |
|---|---|
| `r_task` | Whether the path is task-valid |
| `r_user` | Whether the path passes the personalized verifier |
| `r_pref` | Zero-centered trajectory-level personalization evidence in `[-1, 1]` |
| `residual_log_ratio` | Log ratio between same-user and cross-user trajectory similarity |
| `user_score` | Weighted personalized score |
| `leaf_weight` | Support weight used during reward propagation |
| `user_score_components.target` | Exact target-action matching |
| `user_score_components.order` | Action-order similarity |
| `user_score_components.habit` | Agreement with the user's earlier behavior |
| `user_score_components.avoid` | Preference for avoiding cross-user dependence |

## Advantage Record

`papo_advantages.jsonl` contains one row per candidate action at a tree node.

```json
{
  "tree_id": "...",
  "node_id": "node_00000",
  "action": "select:加号",
  "source": "same_user",
  "r_task": 1.0,
  "r_user": 0.7,
  "a_task": 0.1,
  "a_user": 0.3,
  "uncertainty": 0.2,
  "a_user_conservative": 0.28,
  "a_delta": 0.18
}
```

| Field | Meaning |
|---|---|
| `r_task`, `r_user` | Propagated task and personalized rewards |
| `q_task`, `q_user` | Candidate action values |
| `a_task`, `a_user` | Values relative to the node mean |
| `uncertainty` | Penalty based on support and reward spread |
| `a_user_conservative` | Personalized advantage after uncertainty penalty |
| `a_delta` | Personalized improvement relative to task advantage |

`a_delta` is the primary signal used to create DPO preference pairs.

## DPO Pair Record

`papo_dpo_pairs.jsonl` is the final preference-training dataset.

```json
{
  "node_id": "...",
  "step_id": "...",
  "user_id": "1",
  "episode_id": "...",
  "positive_action": "select:加号",
  "negative_action": "select:其他按钮",
  "advantage_gap": 0.35,
  "weight": 1.75,
  "positive_source": "same_user",
  "negative_source": "cross_user"
}
```

| Field | Meaning |
|---|---|
| `positive_action` | Preferred personalized action |
| `negative_action` | Less preferred action at the same state |
| `advantage_gap` | Difference in `a_delta` |
| `weight` | Training weight derived from the gap |

Pairs with a gap smaller than `margin` are excluded.

## Listwise Target Record

`papo_listwise_targets.jsonl` contains the closed-form target policy for every
offline tree node. Each candidate stores:

- `base_policy_probability`: support-based offline proxy for `pi_0(a|n)`
- `target_policy_probability`: normalized `pi_star(a|n)`
- `a_delta`: conservative residual personalized utility
- `step_id`, `prefix_actions`, and `state_key`: the node-specific proxy context

The LLaMA-Factory export expands every node into one supervised row per
candidate action. `papo_listwise_weight` equals
`target_policy_probability`. With `use_papo_listwise: true`, the Trainer
minimizes:

```text
-sum_a pi_star(a|n) log pi_theta(a|n,H_u)
```

With the default `evidence_transform: tanh_log_ratio`, positive `r_pref`
indicates that a leaf is more similar to same-user history, negative `r_pref`
indicates stronger cross-user similarity, and zero is neutral or inactive.
The legacy `sigmoid_log_ratio` transform remains available for reproducing
older runs.

## Audit and Summary Files

### `papo_raw_audit.json`

Reports raw-to-step conversion quality:

- episode, step, and user counts
- valid observation rate
- next-state coverage
- confident action-grounding rate
- common apps and actions
- warnings and build-selection settings

### `papo_tree_summary.json`

Reports counterfactual tree quality:

- number of trees and leaves
- average nodes, leaves, and candidates
- task and user reward rates
- average verifier-component scores
- candidate-source and depth distributions

### `papo_dpo_summary.json`

Reports DPO export configuration and counts:

- `num_advantage_rows`
- `num_pairs`
- `margin`
- `tau_m`
- `w_max`

### `global_dataset_audit.json`

Reports extraction readiness and official split coverage:

- complete and incomplete episode counts
- available user count
- survey/action mismatches
- train, suggestion-test, execution-test, and total coverage
- whether a full build is ready

Do not treat an audit generated while extraction is actively changing the
directory as a stable snapshot. Run the audit again after extraction finishes.

## Join Keys

```text
task.metadata.papo_episode_id
  = papo_episodes.episode_id
  = papo_steps.episode_id
  = papo_trees.episode_id
  = papo_advantages.episode_id
  = papo_dpo_pairs.episode_id

task.metadata.papo_root_step_id
  = papo_steps.papo_step_id
  = papo_trees.root_step_id
```

These keys allow an execution task to be traced from the original episode all
the way to its final DPO pairs.

## Recommended Validation Rules

Before training, verify:

1. Every history/reference timestamp is strictly earlier than the target time.
2. Every `papo_trees.root_step_id` exists in `papo_steps.jsonl`.
3. Every DPO pair compares two different actions from the same node.
4. `r_task`, `r_user`, `q_task`, and `q_user` remain in `[0, 1]`.
5. Final training inputs exclude all fields under `target`.
6. Smoke artifacts are excluded from final datasets.
